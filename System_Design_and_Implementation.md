# 智颜方正—智能图像矫正与风格男妆生成系统：系统设计与实现文档

## 1. 系统概述
本系统是一款专为男性设计的智能图像处理应用，集成了人脸检测、智能矫正、模糊度评估以及风格化男妆生成等功能。系统旨在帮助用户快速修正自拍照片中的姿态问题，并提供自然得体的虚拟试妆体验。

---

## 2. 系统架构
本系统采用经典的前后端分离架构：
- **前端 (Frontend)**: 基于 Vue 3 + Vite 构建，使用 TypeScript 开发，提供现代化的响应式用户界面。
- **后端 (Backend)**: 基于 Python Flask 框架，集成 OpenCV、MediaPipe、PyTorch 等库，处理图像算法和业务逻辑。
- **数据存储**: 使用 SQLite 数据库管理图片上传记录、处理结果及调试数据；文件系统用于存储实际的图片文件。

---

## 3. 前端实现 (Frontend)

### 3.1 页面路由与功能 (`src/views`)

前端应用包含以下核心页面，由 `src/router/index.ts` 进行路由管理：

1.  **登录页 (`Login.vue`)**
    *   **功能**: 用户凭证验证，进入系统的入口。
    *   **路由**: `/login`
    *   **特点**: 包含交互式动画（如气泡背景、滑动抽屉），提供沉浸式体验。

2.  **首页 (`Home.vue`)**
    *   **功能**: 系统主控台，展示系统核心功能入口及风格展示。
    *   **路由**: `/` (需要登录)
    *   **特点**: 展示三大核心妆容风格（自然清透、轻熟职场、韩系潮流），提供直观的视觉引导。

3.  **上传页 (`UploadView.vue`)**
    *   **功能**: 图片上传界面。
    *   **路由**: `/upload` (需要登录)
    *   **逻辑**: 支持拖拽上传，前端会对图片格式进行初步校验，上传成功后自动跳转至结果处理页。

4.  **结果展示页 (`ResultView.vue`)**
    *   **功能**: 核心交互页面，展示处理流程的各个阶段结果。
    *   **路由**: `/result` (需要登录)
    *   **交互**:
        *   显示原图、矫正后图片及上妆效果图。
        *   提供上妆风格切换按钮。
        *   调用后端压缩 API 进行图片下载。
        *   包含 "模糊检测" 的反馈信息。

5.  **商品详情页 (`Shop.vue`)**
    *   **功能**: 推荐与妆容风格匹配的化妆品。
    *   **路由**: `/shop`
    *   **特点**: 展示相关男士美妆产品。

### 3.2 核心组件 (`src/components`)
为了提高代码复用性，系统封装了多个通用组件：
*   **`ImageUpload.vue`**: 封装了文件选择、拖拽上传及预览逻辑。
*   **`ImageComparison.vue`**: 提供图片处理前后的对比视图。
*   **`ParameterPanel.vue`**: 用于调节图像处理参数的控制面板。
*   **视觉特效组件**: `GalaxyBackground.vue`, `ParticleBackground.vue`, `ModernBackground.vue` 用于增强页面视觉效果。

## 4. 后端实现 (Backend)

后端服务入口为 `backend/app.py`，主要提供 RESTful API 接口。

### 4.1 核心 API 接口
1.  **`/process_upload` (POST)**:
    *   **功能**: 处理用户上传的原始图片。
    *   **流程**: 保存原图 -> 人脸检测 -> 模糊检测 -> 人脸矫正 -> 返回矫正结果 URL。
    *   **数据库操作**: 记录上传、调试 ROI、矫正结果。

2.  **`/apply_makeup` (POST)**:
    *   **功能**: 对矫正后的图片应用指定风格的妆容。
    *   **参数**: `filename` (输入图片), `style` (妆容风格)。
    *   **流程**: 加载模型 -> 生成妆容 -> 保存结果 -> 返回图片 URL。

3.  **`/compress` (POST)**:
    *   **功能**: 图片压缩服务。
    *   **参数**: 图片文件, `quality` (质量)。

### 4.2 核心算法模块 (`backend/*.py`)

1.  **人脸检测 (`face_detection.py`)**
    *   **算法**: 基于 MediaPipe Face Mesh。
    *   **功能**: 检测 468 个人脸关键点，返回人脸边界框及关键点坐标。

2.  **模糊检测 (`blur_detection.py`)**
    *   **算法**: 拉普拉斯方差 (Laplacian Variance)。
    *   **策略**: 结合人脸关键点，提取眼睛、嘴巴等关键区域 (ROI) 进行单独清晰度评估，比全图评估更准确。

3.  **智能矫正 (`face_correction.py`)**
    *   **原理**: 计算双眼连线与水平线的夹角。
    *   **实现**: 计算旋转矩阵，对图像进行反向旋转。
    *   **优化**: 采用无损旋转扩展（Border Replicate），避免旋转后出现黑边或裁剪掉重要内容。

4.  **风格男妆生成 (`makeup_gan.py`)**
    *   **算法**: 基于生成对抗网络 (GAN) 的风格迁移技术。
    *   **演示模式**: 为了保证演示稳定性，集成了 `Demo Mode`，针对特定测试图片直接加载高质量预设结果。
    *   **风格**: 支持 Clean (自然), Business (职场), Idol (潮流) 等多种风格。

5.  **背景去除 (`process_refined.py`)**
    *   **算法**: 精细化漫水填充 (Refined FloodFill)。
    *   **用途**: 针对特定图片（如 Logo）去除纯色背景。

### 4.3 数据管理 (`backend/db_manager.py`)
系统使用 SQLite (`backend/database/app_data.db`) 管理所有数据流转：
*   **`uploads`**: 记录所有用户上传的原图。
*   **`debug_images`**: 存储模糊检测过程中的局部截图，便于算法调试。
*   **`corrected_images`**: 存储自动矫正后的图片记录。
*   **`makeup_images`**: 存储最终生成的妆容效果图。

文件存储结构规范：
```
backend/database/
  images/
    uploads/   # 原图
    outputs/   # 处理结果 (矫正、美妆)
  debug_rois/  # 调试用 ROI
```
