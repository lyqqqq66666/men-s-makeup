# 【Code with SOLO】用SOLO在2天内完成智能男妆系统的数据补齐与后端完善

---

## 摘要

使用 TRAE SOLO 的多 Agent 并行能力，在 2 天内完成了「颜选MenX——男士个性化试妆与美妆智选平台」智能男妆系统的商品数据补齐（184 SKU）、后端代码检查、PRD 文档撰写，并成功备份到 GitHub。原本需要 1-2 周的工作，通过 SOLO 的 AI 辅助在数小时内完成核心部分。

---

## 背景

我是一名大三学生，正在参加数字图像处理课程项目。我们团队开发的「颜选MenX——男士个性化试妆与美妆智选平台」是一款面向 18-35 岁男性的智能图像矫正与风格男妆生成系统，已经完成了前端 Vue 3 和后端 FastAPI 的基础开发。

**面临的挑战**：
- 商品数据严重缺失（只有约 30 个 SKU，需要 150+）
- 后端代码需要检查和优化
- Trae 挑战赛截止日期就在明天（5月10日）
- 需要撰写完整的项目文档

---

## 实践过程

### 1. 任务拆解

我首先让 SOLO 分析了项目现状，然后制定了多 Agent 并行执行方案：

```
┌─────────────────────────────────────────────────────────────┐
│                    主控 Agent (Main)                         │
│         负责协调、代码整合、PRD撰写、最终提交                │
└───────────┬─────────────────────┬───────────────────────────┘
            │                     │
    ┌───────▼───────┐     ┌───────▼───────┐
    │  Agent A      │     │  Agent B      │
    │  数据Agent    │     │  后端Agent    │
    │               │     │               │
    │ • 生成种子数据│     │ • 探索项目    │
    │ • 整理色号    │     │ • 收集信息    │
    │ • 数据入库    │     │ • 返回报告    │
    └───────────────┘     └───────────────┘
```

### 2. SOLO 能力使用

**Agent A — 数据生成**：
- 让 SOLO 读取项目数据库结构（23 张表）
- 基于男士化妆品色号标准生成种子数据
- 输出 `seed_products.py`（162 SKU）+ `makeup_palettes.json`

**Agent B — 项目探索**：
- 并行探索前端路由、后端 API、数据库结构
- 返回完整的功能清单和技术架构报告

**主控 Agent — 文档撰写**：
- 整合两个 Agent 的产出
- 撰写 PRD 产品需求文档
- 准备社区发帖内容

### 3. 关键 Prompt

**数据生成 Prompt**：
```
你是数据Agent，负责为颜选MenX——男士个性化试妆与美妆智选平台项目生成商品种子数据。

任务：
1. 读取数据库结构，了解表结构
2. 基于男士化妆品色号体系，生成种子数据
3. 覆盖5大品类：base(底妆)、brow(眉部)、eye(眼部)、lip(唇部)、contour(修容)
4. 每个品类至少30个SKU，总计150+SKU
```

**项目探索 Prompt**：
```
请帮我探索项目，收集以下信息用于撰写PRD：
1. 读取 src/router/index.ts 了解前端路由和页面结构
2. 读取 backend_new/fastapi_app.py 了解所有API端点
3. 读取数据库文档了解表结构
```

### 4. 踩过的坑

1. **Git 认证问题**：SSH 连接失败，改用 HTTPS + Personal Access Token 解决
2. **MediaPipe 版本问题**：安装 0.10.9 版本解决兼容性问题
3. **.gitignore 配置问题**：原配置把后端代码都忽略了，需要修改

## 效果与总结

### 提效分析

| 任务 | 传统方式 | SOLO 辅助 | 提效 |
|------|----------|-----------|------|
| 商品数据生成 | 3-5 天 | 15 分钟 | **95%+** |
| PRD 文档撰写 | 4-6 小时 | 30 分钟 | **85%+** |
| 项目结构分析 | 2-3 小时 | 5 分钟 | **95%+** |
| Git 备份 | 30 分钟 | 5 分钟 | **83%+** |

### 思考与收获

1. **多 Agent 并行是关键**：让数据生成和项目探索同时进行，大幅节省时间
2. **先规划再执行**：Harness Engineering 框架帮助明确任务边界，避免返工
3. **AI 不替代决策**：关键决策（如 Git 认证方式、数据优先级）仍需人工确认
4. **Done is better than perfect**：时间紧迫时，先完成核心功能，再迭代优化
