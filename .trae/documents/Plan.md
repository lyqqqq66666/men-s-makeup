## 3. Agent执行规范

### 3.1 Agent A 执行指令

```
你是数据Agent，负责为颜选MenX——男士个性化试妆与美妆智选平台项目补齐商品数据。

任务：
1. 读取 /sessions/.../ZhiYan_split_frontend_backend_catalog.xlsx，解析其中的商品数据
2. 基于Agent.md中定义的色号体系，生成男士化妆品种子数据
3. 编写 seed_products.py 脚本，将数据导入SQLite数据库
4. 覆盖5大品类：base(底妆)、brow(眉部)、eye(眼部)、lip(唇部)、contour(修容)
5. 每个品类至少30个SKU，总计150+SKU
6. 每个SKU必须包含：sku_id, spu_id, shade_name, hex_color, render_hex, 
   render_mode, finish_type, opacity, feather, apply_area, category, 
   price, season_match, brand

数据来源：
- Excel中已有的商品数据
- 男士化妆品常见品牌（杰威尔、曼秀雷敦、欧莱雅男士等）
- Agent.md中定义的色号标准

输出：
- backend_new/data/seed_products.py
- backend_new/data/makeup_palettes.json
```

### 3.2 Agent B 执行指令

```
你是后端Agent，负责完善颜选MenX——男士个性化试妆与美妆智选平台项目的后端功能。

任务：
1. 检查 backend_new/ 目录下的所有Python文件，修复明显Bug
2. 优化 excel_importer.py，确保能正确导入Excel商品数据到数据库
3. 检查推荐接口 /api/recommend/products 和 /api/recommend/bundles 的逻辑
4. 确保 /api/makeup/apply 接口的局部试妆功能正常
5. 更新 fastapi_interface_docs.md 文档

注意事项：
- 不要修改前端代码
- 不要修改数据库schema
- 保持现有API兼容性
- 修复时添加注释说明

输出：
- 修复后的 backend_new/*.py
- 更新的 fastapi_interface_docs.md
```
